# The Four Claude Features That Turn AI From Toy to Tool

## Context

A Substack post by Stephen Smith (Intelligence by Intent, March 2026) answering a question he gets constantly from firm leaders: "What's the difference between Skills, Plugins, Projects, and Artifacts in Claude, and when would I actually use each one?" Written for law firm leaders but the framing generalizes to any team adopting Claude.

**The analogy that carries the piece: Claude is a very smart first-year associate.** Passed the bar, can research, write, reason, but day one they don't know your formatting conventions, templates, or how your group organizes things. Capable but useless. Four things make them productive fast, and they map directly to how Claude works:

- **Skills = the practice group memos you hand them.** Reusable workflow instructions written in plain English (Markdown). Claude picks them up automatically when a task fits; they persist across every conversation. No more copy-pasting the same instructions into a new chat every Monday. Anthropic ships built-ins plus a partner directory (Notion, Figma, Atlassian), but the real power is custom ones.
- **Plugins = the entire cookbook plus the kitchen tools.** Bundle Skills together with MCP connectors, slash commands, and sub-agents into one installable package. Install one and Claude shows up already knowing how your practice runs. Anthropic open-sourced 11 at launch in January 2026, expanded across sales, legal, marketing, finance, support, product; a marketplace launched February 2026. A team can package its research protocols, review process, templates, and connectors into a single Plugin; a new hire installs it and Claude knows the house style before orientation ends.
- **Projects = walled-off workspaces per matter.** Context, files, and conversation history isolated to one body of work. Without them everything bleeds together; for a law firm that's a confidentiality problem, full stop. Like an ethical wall for your AI. Team/Enterprise plans add permissions (who can use vs edit).
- **Artifacts = the things it actually produces.** Actual files, not text in a chat window: documents, decks, comparison charts, spreadsheets, interactive tools, small web apps. Preview, edit, download, share via link. The author calls it possibly the most useful feature day-to-day and the one everyone forgets to mention.

**The one-line summary per feature:** Skills are about consistency. Plugins are about capability. Projects are about separation. Artifacts are about output.

**Why leadership should care:** taken together, Claude stops being a chatbot and becomes a configurable work environment: knows your standards, has practice-specific tools loaded, keeps matters isolated, produces actual deliverables. People spend less time on formatting, first drafts, and template wrangling, more on judgment and strategy. Advice: start small, pick one workflow that eats associate time weekly, build a Skill for it, see what happens, then build another one.

Angle note for channels: Altaf runs this exact stack daily as an engineer (custom skills, plugins, isolated project contexts, artifacts), so a developer-flavored translation of the lawyer framing is natural. The "most firms have not configured any of them" gap is the hook: the features are free and sitting unused.
