# Post Plan — claude-four-building-blocks

## Core Insight
Claude stopped being a chatbot the moment you configure its four building blocks: Skills (consistency), Plugins (capability), Projects (separation), Artifacts (output). Most people have configured none of them, so they're running a work environment as a toy.

## Target Audience
Developers and tech professionals who use Claude daily but only through the chat box; team leads deciding how to roll AI out beyond "everyone has a subscription".

## Structure
1. The gap: people use Claude every day and have configured none of its four building blocks
2. The mental model: Claude is a very smart new hire, capable but useless on day one until you give it four things
3. The four blocks, one line each: Skills teach it how you work (consistency), Plugins kit it out for a job (capability), Projects wall off contexts (separation), Artifacts are what it actually produces (output)
4. Practical start: pick one workflow that eats time weekly, build a Skill for it, then build another

## CTA
Open question: "Which of the four are you actually using?" Source link in first comment.

## Tone Range
Educational, conversational, slightly provocative about the unused-features gap.
