8 Claude Code habits worth stealing, curated from a developer's 25 lessons after months of shipping features.

1. Write the spec before opening Claude. Planning is 80 percent of success. AI amplifies clarity or confusion, your choice.

2. One feature per chat, always. Mixing features is coding drunk.

3. Feed it real context: screenshots, file structures, schemas, API docs. Screenshots alone carry 10x more context than text.

4. Git commit after EVERY working feature. Reverting beats fixing.

5. Never accept "should work". Loop the tests until they actually pass.

6. After every completion, ask: "Review your work and list what might be broken."

7. For bug fixes: "Fix this without changing anything else." That one sentence prevents cascade failures.

8. Keep a DONT_DO.md of past failures. The AI forgets between sessions. You shouldn't.

The pattern across all 25: none of them are prompting tricks.

Clear specs. Small scopes. Honest verification. Version control. The discipline that always mattered, now with a multiplier attached.

And the meta-rule that governs the rest: if you're confused, the AI is too. Clarify for yourself first.
