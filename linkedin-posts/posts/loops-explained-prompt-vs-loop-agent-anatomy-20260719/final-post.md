## Final Post

If you're still prompting your coding agent, you're using AI the slowest way there is.

Type a request. Wait. Fix it. Ask again. Every step runs through you. You're the engine, and the moment you stop pushing, everything stops.

The engineers getting the most out of AI made one shift: they stopped prompting agents and started designing loops.

A prompt is an instruction. A loop is a goal the AI keeps working toward until it gets there: plan, do, verify, iterate.

But a loop is only real if it has three organs.

The verifier is the heart. A test, a build, a measurable condition. Without a gate that can fail the work, the agent just agrees with itself on repeat.

State is the memory. What's done, what failed, what's next. Otherwise it repeats the same mistake forever.

The stop condition is the sanity. Success, or a hard limit like "8 tries, then report". Otherwise you built a machine that spends money all night producing nothing.

Before you build one, four questions. Does the task repeat at least weekly? Can something automatically reject bad output? Can the agent do it end-to-end? Is "done" objective? Miss one, keep the manual prompt.

I run my own content pipeline exactly this way: skills wrapped in loops on a schedule, a separate reviewer agent as the gate, a ledger file as state. I review outputs, not process.

Are you still prompting your agents, or designing loops yet?
