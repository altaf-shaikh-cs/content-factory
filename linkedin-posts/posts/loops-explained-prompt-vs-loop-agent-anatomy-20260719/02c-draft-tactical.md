The difference between a prompt and a loop, and how to build your first one without burning tokens all night.

A prompt: one instruction, one answer, then it waits for you.
A loop: a goal the AI works toward until it's met. Plan, do, verify, iterate. You step out; the work keeps going.

A real loop needs three organs:

1. A verifier. A test, a build, a measurable condition. This is the heart. No gate means the agent grades its own homework, and it's a generous grader.
2. State. A small record of what's done, what failed, what's next. This is what makes tomorrow's run resume instead of restart.
3. A stop condition. Success, or a hard cap ("8 tries, then stop and report"). This is what keeps it from running all night for nothing.

Should you even build one? Four questions, all must be yes:
- Does the task repeat at least weekly?
- Can something automatically reject bad output?
- Can the agent do the work end-to-end?
- Is "done" objective, not a judgment call?

Miss one box: keep it as a manual prompt.

And the build order that survives production:
1. Get ONE manual run reliable.
2. Save the instructions as a skill.
3. Wrap the skill in a loop with the gate and the stop.
4. Only then put it on a schedule.

I run my content pipeline this way: scheduled loops, a separate reviewer agent as the gate, a markdown ledger as state. It works while I sleep. I review outputs, not process.
